# Issue Notes

Use this directory only for task notes that are worth keeping in the repository.
Most agent work should proceed directly from the user request and local
evidence.

## Rules

1. Keep notes short and task-specific.
2. Use stable lowercase filenames without timestamps or version suffixes.
3. Do not create GitHub issues, branches, commits, or pull requests unless the
   user explicitly asks.
4. Delete resolved scratch notes; Git history is the archive.

Use `issue_template.md` or `task_template.md` only when a written plan is useful.
